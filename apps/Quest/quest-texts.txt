  СИСТЕМНЫЕ МЕХАНИКИ
═══════════════════════════════════════════════════════════════════

  БОЕВАЯ СИСТЕМА
  ─────────────
  • Каждый раунд: герой атакует одного врага, затем все живые
    враги контратакуют (если герой не убил цель с одного удара).
  • Атака: d20 + модификатор. Попадание: результат ≥ КД цели.
  • Крит (d20=20): автоматеческое попадание.
  • Провал (d20=1): автоматеческий промах.
  • Урон: бросок кости оружия. Воин/Варвар: +1 к урону.
  • Рукопашная (без оружия): 1к2.

  АРБАЛЕТ В БОЮ
  ─────────────
  • Если арбалет экипирован (isCrossbowEquipped), первая атака
    в бою — всегда выстрел (авто, тратит болт). Урон 2к4 + Ловк.
  • После выстрела: флаг crossbow_used блокирует повторный
    выстрел до смены оружия или пропуска хода.
  • Если арбалет в руках, но выстрел уже был: атака unarmed
    1к2 с −2 к броску (арбалет мешает в ближнем бою).
  • crossbow_used сбрасывается при: startCombat, startPatrolCombat,
    passTurn (пропуск хода).

  ДВУРУЧНОЕ ОРУЖИЕ
  ─────────────────
  • Фесс (двуручник, heroId=fess): оружие в обе руки, урон
    складывается (бросаются кости обоих оружий), без штрафа.
  • Все остальные герои: при оружии в обеих руках −3 к броску
    атаки (dual-wielding). Урон складывается.
  • Арбалет (slot: 'both'): занимает обе руки. При экипировке
    в одну руку вторая очищается. Нельзя экипировать предмет
    в руку, если другая занята двуручным (сервер проверяет).
  • UI: locked-слот показывает «занято (Арбалет)» с красной рамкой.

  СМЕНА ОРУЖИЯ В БОЮ
  ───────────────────
  • Только через инвентарь (слоты в боковой панели: →Л / →П).
  • Экипировка/снятие тратит ход — сервер вызывает
    resolveCombatRound(session, true) (игрок не атакует,
    враг атакует).

  АТАКА ИЗ ТЕНИ (shadow strike)
  ──────────────────────────────
  • Флаг shadow_strike_ready ставится при выборе скрытных
    действий (красться, спрятаться, проскользнуть).
  • Только для Плута и Монаха (проверка в applyOutcome).
  • Для всех остальных классов флаг удаляется (не даёт бонус).
  • Первое попадание в следующем бою: +1к6.
  • Патруль не триггерится при shadow_strike_ready.
  • surprise_dc-атаки также ставят shadow_strike_ready
    при успехе (для Плута/Монаха).

  ВНЕЗАПНОСТЬ (surprise_dc)
  ──────────────────────────
  • Если выбран surprise-бой: бросок d20 + Ловк ≥ surprise_dc.
  • Успех: первый враг убит до начала боя (hp = 0).
  • Если все враги убиты — мгновенная победа, контратаки нет.

  ЯРОСТЬ (варвар)
  ────────────────
  • POST /api/sessions/:id/ability { abilityId: 'rage' }
  • Активирует флаг rage_active: +1к4 к урону на следующий удар.
  • 1 раз за квест (флаг rage_used). Кнопка в UI.

  ПОРАЖЕНИЕ (НЕ Game Over)
  ─────────────────────────
  • При hp ≤ 0: герой теряет сознание.
  • Применяется onLose (хил + флаги).
  • Инвентарь конфискуется полностью:
    left_hand = null, right_hand = null, body = null, bag = [], ammo = 0.
  • Флаги прогресса сохраняются (узлы, alert, флаги сюжета).
  • Герой возвращается в узел 1 (камера).
  • Показывается диалог с текстом поражения.
  • После нажатия «Продолжить» — возврат в камеру.

  ПЕРЕЗАПУСК (restart)
  ─────────────────────
  • POST /api/sessions/:id/restart
  • Увеличивает defeatCount на 1.
  • При defeatCount ≥ 3: полный game over (сессия удаляется).
  • Создаётся новая сессия с тем же героем и defeatCount.
  • Старый инвентарь не переносится.
  • Куки/сохранения нет — сессии в памяти сервера.

  ПАТРУЛЬ И ТРЕВОГА (alert)
  ─────────────────────────
  • alert — счётчик, растёт при шумных действиях (обыски,
    выбитые двери, громкие бои).
  • После каждого перехода между узлами (goto, roll,
    continue-dialog) — бросок на патруль:
    d20 + alert + модификатор класса.
  • Модификатор класса: Плут/Монах = −2, Варвар = +2,
    остальные = 0.
  • Сл 15: при ≥ 15 патруль не сработал, при < 15 — стартует бой.
  • Патруль НЕ срабатывает: на узлах 1 и 2 (безопасные);
    при shadow_strike_ready; если в целевом узле есть
    scripted combat-choice.
  • Состав патруля:
    - roll 15–19: 1× Гоблин (7 HP, КД 12, 1к4)
    - roll 20–24: 2× Гоблина
    - roll ≥ 25: 1× Орк (15 HP, КД 13, 1к6+2)
  • Если alert ≥ 30: на следующем переходе Эльфийский страж
    (20 HP, КД 14, 1к8) вместо обычного патруля (TODO).

  ДИАЛОГОВЫЙ ОВЕРЛЕЙ
  ───────────────────
  • Выборы с result.text показывают текст в модальном оверлее
    с кнопкой «Продолжить».
  • Сервер создаёт pendingGoto { target, result, choiceText }.
  • Клиент: POST /continue-dialog → применение outcome,
    проверка патруля.

  ЖУРНАЛ
  ──────
  • Нижний фрейм: последние 5 записей, свежие сверху (reverse).
  • Модалка «Журнал»: все 30 записей (slice(-30)).
  • Каждый выбор игрока логируется как
    «[узел:choiceId] текст выбора».
  • Цветовое кодирование (CSS-классы):
    - journal-item (синий): предметы
    - journal-heal (зелёный): лечение
    - journal-damage (красный): урон вне боя
    - journal-combat (оранжевый): боевые раунды
    - journal-debug (серый, мелкий): только для debug-ленты
  • Debug-лог (_debug[]) отдельно от журнала, не шлётся клиенту.
    Доступен через GET /api/sessions/:id/debug.

  ИНВЕНТАРЬ
  ─────────
  • 6 слотов в сумке (MAX_BAG = 6).
  • При переполнении: предмет теряется, запись в журнал:
    «Сумка переполнена — [предмет] теряется в темноте.»
  • Аммуниция (crossbow_bolts): не в сумку, а в счётчик ammo.
  • Слоты экипировки: левая рука, правая рука, тело.
  • Арбалет (both): при экипировке в одну руку вторая очищается,
    другая блокируется (locked). Нельзя надеть одноручное оружие
    в заблокированную руку.
  • Снятие экипировки (unequip): предмет → сумка (если есть место).
  • Выброс (drop): предмет удаляется безвозвратно.

  СИСТЕМА ПОРАЖЕНИЙ (3 попытки)
  ─────────────────────────────
  • Каждая сессия имеет 3 попытки на прохождение квеста.
  • Поражение засчитывается при:
    — обнулении HP в бою (hero.hp ≤ 0);
    — выборе с target: 0 (смертельный исход);
    — провале броска с outcome.target: 0.
  • Каждое поражение увеличивает счётчик defeatCount.
  • При defeatCount ≥ 3:
    — сессия переходит в состояние gameOver;
    — показывается экран «Вы избиты, обессилели и провалились
      в забытьё. Три попытки исчерпаны — тюрьма оказалась сильнее.»;
    — кнопка «← Назад» ведёт на https://d21-club.ru/;
    — кнопка «Выбрать другого героя» возвращает к выбору.
  • После 3-го поражения все POST-эндпоинты возвращают 400 Game over.
  • Перезапуск через restart не добавляет 1 к defeatCount — счётчик
    уже был увеличен при самом поражении. При defeatCount ≥ 3 restart
    также возвращает game_over.
